## 주제 (1문장)

**수입·국내 길이 단위가 한 발주 건 안에서 섞일 때, 견적·발주 전 치수를 하나의 기준으로 맞추고 판정하는 일.**

---

## R-G-I-O

### R — Reality (현실)

소형 가구 공방·리셀 실무. 카탈로그·인보이스는 **feet/yard/inch**, 발주서·택배·재단은 **m/mm**. 메모는 **"6"**, **"2.5"**처럼 **단위 없이** 적는 경우가 잦음.

**Mom Test 근거**

- 발주서 **"길이(m)"** 칸 vs 카탈로그 **"2.5 yard / roll"**
- 수입 포함 주문 **22건** 중 **3건** feet + yard + inch + meter **동시** 필요
- `UnitConverter.py` 현재: meter/feet/yard만, **1회 1입력**, inch **미지원**

---

### G — Goal (목표)

**견적·발주·발송 판정 전**에 필요한 치수를 **한 기준(m 또는 mm)** 으로 맞춰 **맞다 / 틀리다 / 초과**를 내리는 것.

---

### I — Issue (핵심 장애)

단위가 **출처마다 다르고**, 숫자만 옮기거나 **검색·엑셀·프로그램 재실행**으로 환산·교차확인을 **반복**함. **단위 줄을 끝까지 확인하지 않으면** yard↔m 등 **혼동**이 남음.

| 증상 | 과거 사실 (기준선) |
|------|-------------------|
| 단위 혼동 발주 | **2025-11-07** — 2.5 yard → 2.5 m, **약 35만 원** 손해 |
| 다단위 1건 처리 | **2026-05-20** — **1시간 20분** (구글 + 엑셀) |
| converter 재실행 | **2026-05-28** — `feet:12` → 재실행 → `yard:2.5`, **약 7~15분** 추가 |
| 형식·단위 gap | **2026-06-03** — `2.5` → `Invalid format`; **inch** → `Unknown unit` |
| 출력 후처리 | **2026-06-05** — meter→mm **×1000**, **약 5분** |

---

### O — Outcome (바라는 결과)

- **한 건** 안의 여러 치수를 **같은 기준으로 빠르게 대조**할 수 있음
- **단위 확인 없이** 발주·고객 안내로 넘어가지 않음
- 발주 오류·재발주·공기 손실 **재발 없음**

---

Mom Test **과거 수치**를 기준선으로, **측정 가능**하게 정의.

| # | 기준 | 측정 방법 | 기준선 (Mom Test) |
|---|------|-----------|-------------------|
| 1 | **단위 혼동 발주 오류 0건** | 도입 후 **3개월** yard/m·feet/cm **숫자만 복사**로 인한 재발주·정정 **0건** | **2025-11-07**, **약 35만 원** |
| 2 | **다단위 1건 처리 시간 ≤ 20분** | feet+yard(+inch) 동시 필요 건 환산·대조 시간 | **2026-05-20**, **1시간 20분** |
| 3 | **단위 없는 메모에서 판정 가능** | **"6"**, **"2.5"** 등 입력 후 **m/mm 판정값** 도출 (Invalid/Unknown **0회** 또는 단위 선택 **1단계** 이내) | **6/3** `2.5` 실패; 시뮬 **4/18** 6 cm 오안내 |
| 4 | **한 건 연속 처리** | 동일 PDF·발주 **2개 이상** 치수, **재시작 없이** 처리 | **5/28** **2회** 재실행 |
| 5 | **업무 출력 기준과 일치** | **m 소수 2자리** 또는 **mm 정수**, 추가 계산 없음 | **6/5** ×1000 **5분**; **5/28** 반올림 재확인 **3분** |
