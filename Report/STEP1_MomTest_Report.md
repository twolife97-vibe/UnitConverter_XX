## 1. 페르소나

| 항목 | 내용 |
|------|------|
| **이름** | 박지현 |
| **역할** | 소형 가구 제작 공방 실무 |
| **맥락** | 수입 합판·몰딩 스펙은 feet/yard/inch, 국내 자재·배송 치수는 meter/mm |
| **도구 사용** | `UnitConverter.py` — 2026년 5월 말~6월 초 **총 4회** 실행 |

### 코드 기준 앱 동작 (인터뷰 답변 근거)

- 입력 형식: `unit:value` (예: `meter:2.5`)
- 지원 단위: **meter, feet, yard** (inch 미지원)
- 변환 비율: `1 meter = 3.28084 feet`, `1 meter = 1.09361 yard`
- 실행 1회당 입력 1번, meter/feet/yard 3줄 출력 후 종료
- 콜론 없음 → `Invalid format`, 잘못된 숫자 → `Invalid number`, 미지원 단위 → `Unknown unit`

## 2. 인터뷰 요약 (과거 사실)

### 단위 혼재로 곤란했던 경험

- **2026-05-20** 수원 책장 발주: feet / yard / inch / meter **동시** 필요 → **1시간 20분** (구글 + 엑셀)
- **2025-11-07** 카탈로그 **2.5 yard**를 발주서 **길이(m)** 칸에 **2.5**만 입력 → **약 35만 원** 손해 (재발주 + 시공 1일 지연)

### yard를 meter로 잘못 읽은 이유

- PDF에 **"2.5 yard / roll"**, 발주서에는 **"길이(m)"** 칸만 존재
- **숫자 2.5만** 보고 단위 줄 확인 없이 입력
- 국내 자재 3년간 meter 전제 습관

### 업무에서 숫자만 입력하는 패턴

- **feet** (가장 많음), **yard**, **inch** — 수입 자재 카탈로그에서 메모·발주 시 숫자만 먼저 옮기는 경우 빈번
- **meter** — 발주서 칸 이름이 m라 숫자만 넣어도 meter로 간주
- **mm** — 현장 실측 메모에 **"900"**, **"1200"** 등 단위 없이 기록 (주 2~3회)

### converter 사용 경험 (4회)

| 날짜 | 입력 | 결과 |
|------|------|------|
| 2026-05-28 | `feet:12` | 정상 — 구글·엑셀과 일치 |
| 2026-05-28 | `inch:24` | `Unknown unit: inch` |
| 2026-06-03 | `2.5` | `Invalid format` — 엑셀 우회, 지인 답장 10분 대기 |
| 2026-06-05 | `yard:2.5` | 정상 — 엑셀 2.734025 m와 일치 |

- converter **값 오류로 인한 금전 손실 없음**
- **mm·inch 미출력**, **자릿수 고정 없음**, **한 실행당 1입력** → 추가 엑셀·재실행 **5~15분** (5/28, 6/5)

### 한 번에 여러 입력이 필요했던 경우

- **2026-05-28:** `feet:12` → 재실행 → `yard:2.5` → inch는 converter 밖 (엑셀) — **약 15분**
- **2026-06-05:** `yard:2.5` → 재실행 → `feet:8` — **약 6분**
- **2026-05-20** 유형 (단위 3~4개): converter **미사용**, 엑셀·구글 **1시간 20분**

### 네 단위 동시 필요

- 2026년 1~5월 수입 포함 주문 **22건** 중 **3건**에서 feet + yard + inch + meter **동시** 등장

## 4. 진짜 문제 (한 문장)

**수입·국내 자재를 견적·발주할 때 feet/yard/inch/m/mm 단위가 카탈로그·발주서·현장 메모마다 섞여, 숫자만 옮기거나 환산을 반복·교차 확인하다 시간을 쓰고, 단위를 끝까지 확인하지 않으면 발주 오류와 재발주·공기 손실로 이어진다.**
