# UnitConverter_XX — Mom Test 워크북 채점 프롬프트

```
아래 Mom Test 결과를 채점해줘. 수정하지 마.
[워크북 전체]

체크:
- [ ] 미래 가정("~하면 좋겠다") 없음
- [ ] 과거 행동·시간·실수 구체성 있음
- [ ] 진짜 문제에 솔루션名(TDD/PyQt/Cursor) 없음
- [ ] 표면 문제와 진짜 문제가 분리됨
- [ ] UnitConverter_XX 도메인 반영

점수 /10 + 고칠 질문 1개만 제안
```

---

## 채점 기준 (참고)

| 항목 | Pass 조건 |
|------|-----------|
| 미래 가정 | 인터뷰·진짜 문제에 "쓰고 싶다/좋겠다/만들면" 없음 |
| 구체성 | 날짜, 분, 원, 입력값, 순서가 1개 이상 |
| 솔루션名 | 진짜 문제에 앱·프레임워크·도구명 없음 |
| 분리 | 표면=제품 가정 / 진짜=불편·비용·판정·재현 |
| 도메인 | meter/feet/yard, CLI 제약, 발주·환산 맥락 |

## 감점 패턴

- 진짜 문제 한 문장 과부하 (재현 약함)
- 증거 3줄이 여러 사건을 "/"로 합침
- 최대 pain과 converter 사용 전후 연결 없음

## 개선 질문 템플릿

> 「converter를 쓴 **이후**에도, [핵심 실수 패턴]이 **한 번이라도** 있었나요? 있었다면 **언제**, 없었다면 **그때 뭐가 달랐나요**?」
