# UnitConverter_XX — Mom Test 역할 분리 시뮬레이션 프롬프트

```
역할 분리 Mom Test 시뮬레이션 (UnitConverter_XX).

Role 1 — 인터뷰어: Mom Test 규칙 준수, 질문만
Role 2 — 페르소나: "UnitConverter app 을 사용하는 사용자"
  (가상이지만 plausible한 과거 사실만)

3턴 진행:
- Turn 1: 인터뷰어 질문 → 페르소나 답
- Turn 2: 추궁 → 답
- Turn 3: 추궁 → 답

마지막: 표면 문제 vs 진짜 문제 (가상 인터뷰임을 명시)

⚠️ 내 실제 경험과 섞지 말 것. 시뮬레이션 라벨 필수.
```

---

## Role 1 — 인터뷰어 규칙

- **질문만** 한다 (조언·피치·기능 제안 금지)
- **과거·구체·행동** 중심: "언제 / 무엇을 / 몇 분 / 그다음"
- **추궁**은 Turn 2·3에서 시간·순서·비용·대안 행동으로 좁힌다
- 금지: "~하면 좋겠다", "우리 앱 어떠세요", "기능 추가하면"

## Role 2 — 페르소나 규칙

1. `UnitConverter.py` **구현 코드** 동작 기준으로만 답한다
2. 금지: 모호한 답변
3. 금지: 솔루션 섞인 답변 ("~앱 있으면")
4. **과거 사실**만 (날짜·숫자·순서·분·원)

### 코드 동작 (답변 범위)

| 항목 | 내용 |
|------|------|
| 입력 | `unit:value` (예: `meter:2.5`) |
| 지원 단위 | meter, feet, yard |
| 미지원 | inch → `Unknown unit: inch` |
| 형식 오류 | 콜론 없음 → `Invalid format` |
| 실행 | 1회 1입력 → 3줄 출력 → 종료 |
| 비율 | `1 m = 3.28084 ft`, `1 m = 1.09361 yd` |

## Turn 1 추천 질문 (오프닝)

- "단위가 섞여서 업무에 지장이 있었던 **가장 최근** 경우는? **언제**, **무엇**을 하려다 막혔나요?"

## Turn 2 추천 질문 (추궁)

- "그날 converter 쓰기 **전**, **정확히 어떤 순서**로 확인하셨나요? **몇 분**?"

## Turn 3 추천 질문 (추궁)

- "converter 썼을 때 **그 직후** 같은 건을 **어떻게** 이어갔나요? **재실행**? **다른 도구**?"

[시뮬레이션] 진짜 문제 (한 문장, 솔루션名 없음)
