## 2. 인터뷰 역할 설정 프롬프트

```
UnitConverter_XX STEP 1 — Mom Test 인터뷰를 진행할꺼야.
나는 스타트업 멘토(Rob Fitzpatrick Mom Test 스타일)야.
너는 [페르소나: UnitConverter app 을 사용하는 사용자] 역할이야.

규칙:
1. 프로젝트에 구현된 코드를 파악해서 해당 코드 동작 기반으로 질문에 대답한다
2. 금지: 모호한 답변을 하지 않는다
3. 금지: 솔루션 섞인 답변을 하지 않는다
4. 과거 실제 사실에 대한 답변만 한다
```

### 페르소나 컨텍스트 (AI 역할용)

| 항목 | 내용 |
|------|------|
| **이름** | 박지현, 29세 |
| **역할** | 소형 가구 제작 공방 실무 (경력 6년) |
| **맥락** | 수입 합판·몰딩 feet/yard/inch, 국내 meter/mm |
| **도구** | `UnitConverter.py` — 2026년 5월 말~6월 초 **4회** 사용 |

### 코드 동작 (답변 범위)

- 입력: `unit:value` (예: `meter:2.5`)
- 지원: meter, feet, yard
- 비율: `1 m = 3.28084 ft`, `1 m = 1.09361 yd`
- 1실행 1입력, 3줄 출력 후 종료
- inch 미지원, JSON/CSV 없음, 설정 파일 없음

## 3. 멘토 인터뷰 질문 목록 (실행본)

1. 단위가 섞여서 곤란했던 적이 있나요?
2. yard를 meter로 잘못 읽었다고 하셨는데. 왜 잘못 읽으셨나요?
3. converter에 숫자만 입력했을 때 곤란을 겪은 경우가 있었나요?
4. 그 문제가 일어났을 때 다시 meter:2.5처럼 입력을 하셨나요?
5. 실제 업무시 숫자만 입력하는 경우가 많나요?
6. 그럼 보통 숫자만 입력하는 경우는 주로 어떤 단위일 때인가요?
7. feet, yard, inch, meter가 모두 한꺼번에 필요한 경우도 있나요?
8. converter 값이 잘못되어서 낭패를 본 경험이 있나요?
9. 실제 업무에서 converter가 준 값이 구체적이지 않아 문제가 된 적이 있나요?
10. 한번에 한 입력만 되어서 문제가 된 적이 있나요?
11. 한번에 여러 입력이 필요한 적이 있다고 하셨는데 그때는 어떤 식으로 입력이 필요하셨나요? 그리고 여러 입력이 되지 않아서 그 뒤에 어떻게 하셨나요?

## 5. Mom Test 질문 가이드 (추가 인터뷰용)

### 좋은 질문 (과거·구체·행동)

- "마지막으로 단위 때문에 발주를 틀린 게 언제인가요?"
- "그날 정확히 어떤 순서로 확인하셨나요?"
- "그때 시간·돈이 얼마나 들었나요?"
- "그 이후 뭐가 바뀌었나요?"

### 피해야 할 질문 (의견·미래·칭찬·피치)

- "이런 converter 있으면 쓰실 건가요?"
- "기능을 추가하면 좋을 것 같으세요?"
- "우리 앱 어떻게 생각하세요?"

### Mom Test 3원칙

1. **그들의 삶**에 대해 말하게 한다 — 내 아이디어 말고
2. **과거의 구체적 사실**을 묻는다 — 미래 의견·일반론 말고
3. **적게 말하고 많이 듣는다** — 솔루션을 섞지 않는다
